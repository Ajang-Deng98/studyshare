# Contributing to StudyShare

Thank you for your interest in contributing to StudyShare! This document provides guidelines for contributing to the project.

## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a new branch for your feature/fix
4. Make your changes
5. Test your changes
6. Submit a pull request

## Development Setup

### Using Docker (Recommended)
```bash
docker-compose up -d
```

### Manual Setup
See the main README.md for detailed setup instructions.

## Code Style Guidelines

### Frontend (React/TypeScript)
- Use TypeScript for all new components
- Follow existing naming conventions
- Use Tailwind CSS for styling
- Ensure components are responsive

### Backend (Django)
- Follow PEP 8 style guidelines
- Use type hints where applicable
- Write docstrings for functions and classes
- Follow Django best practices

## Testing

- Write tests for new features
- Ensure all tests pass before submitting PR
- Test on different screen sizes for frontend changes

## Pull Request Process

1. Update documentation if needed
2. Add tests for new functionality
3. Ensure CI pipeline passes
4. Request review from team members
5. Address feedback promptly

## Reporting Issues

When reporting issues, please include:
- Clear description of the problem
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if applicable
- Environment details

## Questions?

Feel free to reach out to the team members listed in the README.md file.
